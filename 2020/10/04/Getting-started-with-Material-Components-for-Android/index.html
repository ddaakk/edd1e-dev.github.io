Getting started with Material Components for Android

Author: Kang Sang Woo Date: October 4, 2020  17:32:42

Material로 이전하기

Material Components로 변경하기

MDC(Material Design Components) 는 기존의 안드로이드의 Design Support Library 를 대체합니다.

기존에 사용하던 AppCompat 템플릿입니다.

서포트 라이브러리에서 제트팩으로 이전하기

MDC 를 사용할 수 있으려면 기존 Support Library 를 Android Jetpack 으로 변경해야만 합니다. Jetpack 은 새로운 androidx 이름 공간을 사용합니다. MDC는 AndroidX 라이브러리로 빌드 되었고 기존에 지원하던 Support Library 와 독립된 버전이기 때문에 이전 Support Library 를 필수로 MDC 로 변경해야 합니다.

AndroidX 로 이전하기 위해서, 안드로이드 개발자 공식 사이트에서는 official developer documentation 나 Android Dev Summit ’19 에서 포스팅한 “Migrating to AndroidX: The time is right” talk 를 보길 추천합니다. 안드로이드 스튜디오는 Design Support 라이브러리의 의존성을 MDC로 리팩토링해줄 겁니다.

메모: 제트팩과 1.0.0 버전의 MDC 아티펙트는 Support Library 28.0.0 아티펙트와 2개가 서로 호환성이 있습니다. 28 버전이 아니라면 먼저 업그레이드 후에 이주할 것을 추천합니다.

MCD 1.0.0 으로 업데이트 하기

만약 안드로이드 스튜디오를 사용하고 있다면 제트펙으로 이전을 위해 ‘Refactor > Migrate to AndroidX’ 툴을 사용하세요. 이 과정을 진행하면 Design Support 라이브러리가 MDC 로 배치되므로 바로 밑에 나올 과정을 생략해도 됩니다.

만약 리팩토링 과정을 하지 않았다면, 직접 아래의 과정을 수행해서 의존성을 추가해야 합니다.

또한 패키지의 Design Support Library 클래스가 참조되는 namespace 들을 android.support.design to com.google.android.material 로 변경해줘야 합니다. (XML 레이아웃과 코드 모두 포함). 옮기기 위해 클래스 매핑 테이블을 확인해주세요.

테마 변경하기

앱 테마를 Material 컴포넌트 테마로부터 물려받아야 합니다. 프로젝트에 적용된 theme overlays 와 추가된 테마들에도 동일한 적용이 필요합니다.

기존에 AppCompat 테마를 사용하고 있었다면 MDC-안드로이드 테마를 사용하기 위해 하나씩 변경해줘야 합니다. 대부분 간단하게 MaterialComponents를 부모로 가진 AppCompat의 일부로 바꿀 수 있습니다.

라이브러리 의존하기

안드로이드를 위한 Material 컴포턴트들은 구글 메이븐 저장소를 통해서 이용할 수 있다.

사용 방법

1) 어플리케이션의 build.gradle 파일을 열어주세요.
2) 구글 메이븐 저장소인 google() 을 repositories 섹션에 포함시켜 주세요.

예시

3) dependencies 섹션에 라이브러리를 추가해주세요.

예시

라이브러리의 최선 버전을 알기 위해 구글 메이븐 저장소 나 MVN 저장소 를 방문해주세요.

새로운 Namespace 와 AndroidX

앱이 현재 기존의 디자인 서포트 라이브러리에 의존하고 있다면 안드로이드 스튜디오에서 제공하는 옵션인 Refactor 를 사용해서 AndroidX를 통해 리팩토링 할 수 있습니다. 앱은 의존성과 코드를 새로운 androidx 패키지와 com.google.android.material 라이브러리를 사용할 것 입니다.

androidx 와 com.google.android.material 패키지로 아직 변경하고 싶지 않다면 com.android.support:design:28.0.0 의존성을 사용하면 됩니다.

메모: com.android.support 과 com.google.android.material 의존성을 앱에서 동시에 사용할 수 없습니다.

안드로이드 10을 사용한 앱 컴파일하기

안드로이드에서 Material Components 사용하기 위해선 Support Libraries 최신 버전이 필요합니다. 안드로이드 10을 사용하기 위해선 안드로이드 스튜디오 3.5 나 더 높은 버전이 필요하며 앱의 compileSdkVersion 을 29로 업데이트 해야 합니다.

AppCompatActivity를 사용 중인지 확인하기

AppCompatActivty 를 사용하면 모든 컴포넌트가 정상적으로 작동합니다. AppCompatActivity 로 상속 받지 않으려면 AppCompatDelegate 를 액티비티에 사용하면 됩니다.

앱 테마를 Material 컴포넌트 테마로 변경하기

앱 전체의 테마를 변경하는 것이 추천하는 방법이지만 기존 레이아웃의 컴포넌트 모양과 동작이 변경될 수 있으니 충분한 테스트가 필요합니다.

참고: 테마를 변경할 수 없는 경우 다음 중 하나를 수행할 수 있습니다.

  • Material Components Bridge 테마 중 하나에서 상속 합니다.
  • AppCompat 테마에서 상속 받은 걸 유지하면서 테마에 몇 가지 새 테마 속성을 추가하면 됩니다.

Material 컴포넌트 테마

  • Theme.MaterialComponents
  • Theme.MaterialComponents.NoActionBar
  • Theme.MaterialComponents.Light
  • Theme.MaterialComponents.Light.NoActionBar
  • Theme.MaterialComponents.Light.DarkActionBar
  • Theme.MaterialComponents.DayNight
  • Theme.MaterialComponents.DayNight.NoActionBar
  • Theme.MaterialComponents.DayNight.DarkActionBar

다음 중 하나를 상속 받도록 앱을 변경해줘야 합니다.

Bridge 테마

테마를 MaterialComponent 로부터 상속 받도록 바꿀 수 없다면 Material Components Bridge 테마로부터 상속 받을 수 있습니다.

Theme.MaterialComponents 와 Theme.MaterialComponents.Light 모두 .Bridge 테마를 가지고 있습니다.

  • Theme.MaterialComponents.Bridge
  • Theme.MaterialComponents.Light.Bridge
  • Theme.MaterialComponents.NoActionBar.Bridge
  • Theme.MaterialComponents.Light.NoActionBar.Bridge
  • Theme.MaterialComponents.Light.DarkActionBar.Bridge

Bridge 테마를 사용하면 앱 테마를 변경하지 않고도 Material 디자인 Components 를 사용할 수 있습니다.
