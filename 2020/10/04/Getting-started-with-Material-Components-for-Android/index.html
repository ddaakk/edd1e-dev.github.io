Getting started with Material Components for Android

Author: Kang Sang Woo Date: October 4, 2020  17:32:42

Material Components로 변경하기

MDC(Material Design Components) 는 기존의 안드로이드의 Design Support Library 를 대체합니다.

기존에 사용하던 AppCompat 템플릿입니다.

MDC 를 사용할 수 있으려면 기존 Support Library 를 Android Jetpack 으로 변경해야만 합니다. Jetpack 은 새로운 androidx 이름 공간을 사용합니다. 그리고 이전 Support Library 패키지들은 MDC는 AndroidX 라이브러리로 빌드 되었기 때문에 필수로 변경해야 합니다.

AndroidX 로 이전하기 위해서, 안드로이드 개발자 공식 사이트에서는 official developer documentation 나 Android Dev Summit ’19 에서 포스팅한 “Migrating to AndroidX: The time is right” talk 를 보길 추천합니다. 안드로이드 스튜디오는 Design Support 라이브러리의 의존성을 MDC로 리팩토링해줄 겁니다.

메모: 제트팩과 1.0.0 버전의 MDC 아티펙트는 Support Library 28.0.0 아티펙트와 2개가 서로 호환성이 있습니다. 28 버전이 아니라면 먼저 업그레이드 후에 이주할 것을 추천합니다.

MCD 1.0.0 으로 업데이트 하기

만약 안드로이드 스튜디오를 사용하고 있다면 제트펙으로 이전을 위해 ‘Refactor > Migrate to AndroidX’ 툴을 사용하세요. 이 과정을 진행하면 Design Support L라이브러리가 MDC 로 배치되므로 바로 밑에 나올 과정을 생략해도 됩니다.

또한 패키지의 Design Support Library 클래스가 참조되는 namespace 들을 android.support.design to com.google.android.material 로 변경해줘야 합니다. (XML 레이아웃과 코드 모두 포함). 옮기기 위해 클래스 매핑 테이블을 확인해주세요.
