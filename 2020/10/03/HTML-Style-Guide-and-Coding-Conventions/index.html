HTML Style Guide and Coding Conventions

Author: Kang Sang Woo Date: October 3, 2020  0:10:23

일관되고 깔끔한 HTML 코드는 더 쉽게 다른 사람들이 읽고 이해할 수 있게 해줍니다.
여기 좋은 HTML 코드를 작성하기 위한 몇 가지 가이드 라인과 팁을 적겠습니다.

Element 이름을 소문자로 사용하기

HTML 은 Elment 이름들을 대문자와 소문자를 섞어서 사용하도록 지원해줍니다.
하지만 W3S 에서는 소문자 태그를 사용하도록 추천합니다. 이유는

  • 대, 소문자를 같이 쓰는 것은 보기 불편합니다.
  • 개발자들은 주로 소문자를 사용합니다.
  • 소문자가 더 깔끔합니다.
  • 소문자가 작성할 때 더 편합니다.

Attribute 이름 소문자로 사용하기

HTML 은 Attribute 이름을 소문자와 대문자를 섞어 쓰도록 허용합니다.
하지만 W3C 는 attribute 이름을 소문자로 사용할 것을 추천합니다. 이유는 다음과 같습니다.

  • 대, 소문자를 같이 쓰는 것은 보기 불편합니다.
  • 개발자들은 주로 소문자를 사용합니다.
  • 소문자가 더 깔끔합니다.
  • 소문자가 작성할 때 더 편합니다.

항상 Attribute 값 참조하기

HTML 은 attribute 값을 참조 없이 사용하도록 허용합니다.
하지만 W3C 는 attribute 값을 참조하도록 추천합니다. 이유는 다음과 같습니다.

  • 개발자들은 보통 attribute 값들을 참조합니다.
  • 참조된 값은 읽기 더 편합니다.
  • 값이 공백을 포함하면 무조건 참조를 사용해야 합니다.

HTML 은 등호 주변의 공백을 허용합니다. 하지만 공백이 없는 것이 읽는 것에 더 편합니다.

긴 코드 줄들 피하기

HTML 에디터를 사용할 때 HTML 코드를 읽기 위해 오른쪽이나 왼쪽으로 스크롤 하는 것은 편리하지 않습니다.
긴 코드 줄을 피하세요.

공백 줄과 들여쓰기

이유 없이 공백 줄, 공백, 들여쓰기를 쓰지 마세요.
가독성을 위해서 크거나 논리적인 코드들을 분리하기 위해 공백 줄을 사용하세요.
가독성을 위해서 들여쓰기로 두 번 SPACE 를 사용하세요. TAB 키를 사용하지 마세요.
